# CSV functinality.

As as Rust software architect do the next changes to calculate the scores provided from a CSV file.

These are the requirements:

- The input CSV file will have 2 columns x and y with headers, the coordinates can be f32 types but the numbers can be expresed as integers too. By exmple 3.0 and 3 are valid inputs.

- The program will create an output CSV file with x, y, and score columns, with header. In case the coordinates create an Error, such score will be set to ERROR.

- The name of the output score CSV file can be provided or not by user. If the name is not provided, take the input name plus a "_score".csv name as default.

- Add the proper parameters and change actual x and y parameters and validations to be compatible with this new functionality.

- Create new unit tests for the CSV file functionality.
